Scrollbar-gutter

Позволяет резервировать место для полосы прокрутки, предотвращая нежелательные изменения макета по мере роста контента, а также избегая ненужных визуальных эффектов, когда прокрутка не нужна.

Применяется только в том случае, если к элементу так же применяется свойство overflow в значениях hidden, scroll, auto.

Поддержка браузерами: 73.66%

Значения свойства:

  • auto: никаких изменений по сравнению с текущим поведением. Значение по умолчанию.
  • stable: добавляет отступ под стандартную полосу прокрутки
  • stable both-edges: добавляет отступ под стандартную полосу прокрутки с симметричным отступом с другой стороны

Inset

Свойство inset заменяет четыре других свойства: top, right, bottom и left , позволяя указать вставку позиционируемого элемента сразу со всех четырех сторон.

Поддержка браузерами: 92.29%

Text-overflow и line-clamp

Свойства text-overflow и line-clamp используются для обрезания текста, если он переполняет элемент. Свойство text-overflow используется для однострочного текста, а свойство line-clamp полезно для обрезания многострочного текста.

Свойства работают в том случае, если к элементу применяется свойство overflow в значениях hidden, scroll или auto.

Поддержка браузерами text-overflow: 92.29%

Поддержка браузерами line-clamp в комбинации со свойством display: -webkit-box: 97.11%

Значения свойства text-overflow:

  • clip: текст обрезается по размеру области.
  • ellipsis : текст обрезается и к концу строки добавляется многоточие.

Для работы свойства так же необходим запрет на перенос строки свойством white-space: nowrap

Свойство line-clamp работает только для блочного контейнера в сочетании со свойством display и -webkit-box-orient. Line-clamp используется с префиксом -webkit, а в качестве значения принимает число строк, на которое будет действовать перенос. 
                  display: -webkit-box;
                  -webkit-line-clamp: 2;
                  -webkit-box-orient: vertical;
                  overflow: hidden;

Content-visibility

Свойство content-visibility предназначено для ускорения первого рендеринга веб-страницы, чтобы ускорить доступность контента пользователю. Оно сообщает браузеру, какие части страницы содержат изолированный контент, что, в свою очередь, позволяет браузеру оптимизировать отрисовку страницы за счет отложенных вычислений.

Поддержка браузерами: 71.89%

У свойства content-visibility 3 значения:

  • visible: значение по умолчанию. Никак не влияет на отображение контента.
  • hidden: скрывает элемент, но сохраняет его состояние рендеринга, т.е. поведение схоже со свойством display: none, но затраты на повторную отрисовку этого элемента намного меньше.
  • auto: ограничивает вычисление размеров, расположения, стилей и отрисовки потомков элемента с этим значением при первой отрисовке.

С помощью свойства content-visibility браузер загружает и стилизует контент, который пользователь в данный момент видит на экране. Для остального контента достаточно указать его высоту. В этом случае в качестве плейсхолдера выступает свойство contains-intrinsic-size.

Самостоятельные значения transform

Свойства scale(), translate(), и rotate(), которые раньше объявлялись как часть значения transform, теперь стали самостоятельными и их можно указывать в правилах отдельно.

Поддержка браузерами: >90%

:has()

Псевдокласс :has() позволяет выбрать элемент, уточнив основной селектор дополнительным. В отличие от привычных селекторов, правило применится только к тому селектору, который был описан до :has(). Использование :has() не влияет на специфичность, т. е. при расчёте веса селектора учитывается только целевой селектор, к которому применяется этот псевдокласс.

Поддержка браузерами: 86.19%

Использование :has() позволяет добавить условия для применения стилей. Его так же можно комбинировать с другими псевдоклассами и селекторами, что, по сути, даёт возможность прибегать к условным операторам И и ИЛИ в правилах стилей

@layer

Порядок загружаемых стилей очень важен, так как стили, загруженные последними, могут перезаписывать ранее загруженные стили.

С появлением @layer можно предварительно определить слои и их порядок, а затем поместить в них загружаемые стили. Это позволит сохранить переопределение стилей, но без контроля за порядком их загрузки.

Поддержка браузерами: 89.9%

Для удобства разработки в Chrome DevTools поддерживается отображение названий и группировка стилей по слоям.

Синтаксис @layer:

  • @layer layer-name {rules}
  • @layer layer-name;
  • @layer layer-name, layer-name, layer-name;
  • @layer {rules}

Где layer-name - название каскадного слоя, а rules - набор правил CSS в каскадном слое.

Любые стили, не описанные в именованных слоях, собираются вместе и помещаются в один анонимный слой. Такие стили, объявленные вне слоя, переопределяют стили, объявленные в слоях, независимо от их специфичности.

Есть 3 способа объявить слой:

В последнем случае, слой не именованный, но объявленный. В дальнейшем ему уже нельзя будет переопределить стили. Приоритет по специфичности у такого слоя, такой же как и у именованных слоёв

Так же уже имеющиеся стили можно определить как слой через @import: 

@import "theme.css" layer(utilities);

Именованным слоям могут быть определены или переопределены стили: 

@layer base, special;

У слоёв объявленных таким образом специфичность устанавливается в обратном порядке - слой, указанный последним, имеет набольший приоритет и переопределяет CSS правила предыдущих слоёв.

Так же слой, может быть вложен в слой:

@layer framework {
                  @layer layout {
                      }
                        }

Чтобы добавить правила к слою внутри framework, его название указывается через .: 

@layer framework.layout {
                  p {
                        margin-block: 1rem;
                          }
                            }
